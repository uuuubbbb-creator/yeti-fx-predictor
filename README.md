# 1. Problem — 여행은 늘었는데 판단 근거는 없다

## 1-1. 일본 여행 시장은 역대 최대, 환율은 역대급 변동성

[📷 일본 찾는 한국인 관광객 수 추이 그래프 / 원·엔 환율 추이 그래프]

최근 일본 여행 시장은 팬데믹·외교 변수에도 불구하고 빠르게 회복해 2015년 400만 명 → 2023년 882만 명으로 역대 최대 방문객을 기록했습니다. 동시에 엔/원 환율은 2023년 7월 100엔당 852.7원에서 2024년 4월 1,006.9원까지 약 **20%** 등락을 반복했습니다. 여행 수요는 폭발하는데 환율은 3년간 높은 변동폭을 유지하며, **같은 100만 원 예산으로 최대 20만 엔 가까운 체감 구매력 차이**가 발생하는 환경이 조성되고 있었습니다.

## 1-2. 여행자의 두 가지 Pain-Point

- **환전 시점 불확실성**: 본격 휴가철을 앞두고 커뮤니티·SNS에는 "언제 환전해야 하나"는 질문과 팁이 넘쳐납니다. 객관적·실시간 정보를 찾기 어려우니 여행객은 감(感)에 의존하거나 '복불복'으로 환전합니다.
- **환전 실행 방식 결정의 어려움**: 엔화 강세가 가팔라지자 카드사와 은행은 "여행객들이 환전 건수를 늘리거나 카드·현금 비중을 조정하는 등 분할·사전·카드 사용 전략을 놓고 혼란을 겪는다"고 분석합니다. 어느 정도를 분할 환전할지, 혹은 일괄 환전이 나은지 판단 기준이 모호합니다.

> **핵심 문제**: 일본 여행 수요 급증과 높은 환율 변동성 속에서, 여행자는 ① '언제' 환전해야 유리한지 판단할 근거가 없고, ② '어떤 방식(일괄·분할·카드)'으로 실행할지 결정하기 어렵습니다. 이 두 Pain-Point는 출국 전 재무 의사결정 스트레스를 키우고, 잘못된 선택 시 여행 총비용을 예측보다 크게 초과시킵니다.

# 2. Research — 국민여행조사로 "누구를 위한 서비스인가"부터 좁히다

## 2-1. 타깃 확정: 일본 여행자, 그리고 그중에서도 2030세대

[📷 EDA 결과 — 가장 많이 방문한 해외 국가 top10 막대그래프 / 일본 여행자 연령 분포 히스토그램]

2024년 국민여행조사 원본(5만 행 이상, 700개 이상 컬럼)의 해외여행 RAWDATA를 EDA한 결과, 전체 방문 국가 중 **일본이 40.4%로 압도적 1위**였습니다. 연령대별로 보면 일본 방문자의 27%가 20대, 23.9%가 30대로 **2030세대가 50.9%**를 차지했습니다. 여기에 한국경제(2024.3) 설문조사에서 30대 여행자의 76%가 '국내 숙박비 부담' 때문에 해외(주로 일본)를 선택했다는 근거를 더해, 서비스의 핵심 타깃을 **가성비에 민감한 2030세대**로 확정했습니다.

## 2-2. 소비 패턴 검증: 절약과 플렉스가 공존하는 세대

[📷 연령대별 1차 여행 1인 지출 비용 분포 박스플롯 / 연령대별 항공여객 지출 비용 / 여행 전후 구매 항목 비중 비율 그래프]

세 가지 EDA 근거로 2030세대의 소비 성향을 구체화했습니다.

1. **지출 편차가 세대 중 최대**: 20-30대는 1인 지출 박스플롯에서 400-500만 원 구간에 이상치가 집중돼 평균·최댓값이 가장 높지만, 중앙값(IQR)은 40-50대와 비슷합니다. 즉 2030세대 전체가 과소비하는 게 아니라, '절약형'과 '플렉스형' 소비 패턴이 공존하며 상황에 따라 지출 구조를 유연하게 조정하는 성향이 강합니다.
2. **필수 비용은 적극적으로 절감**: 20대의 외국 항공사(LCC) 이용률은 36%로 전 세대 중 최고였습니다. 일본 노선 외국 항공사의 80% 이상이 피치·제트스타 같은 LCC임을 고려하면, MZ세대는 동일 노선에서 더 저렴한 운임을 적극적으로 찾아 필수 비용인 항공권을 절감하고 있습니다.
3. **절감한 돈은 경험에 재투자**: 20대의 숙박·교통·식음료 비중은 38.1%로 전 세대 중 최저이며, 절감된 예산은 체험·콘텐츠(기타) 항목에 25.5%로 재투자됩니다.

> **즉 2030세대는 "언제·어디서·얼마나 싸게" 소비할지 고민하면서 동시에 "돈 쓸 가치가 있다면 아끼지 않는다"는 이중적 소비 태도를 보입니다.** 변동성이 큰 엔/원 환율 상황에서 '최적 환전 시점과 방식'을 안내해 필수 비용을 더욱 절감하려는 수요가 이 세대에서 가장 높다는 것이, 서비스가 제공할 실질적 가치와 직접 연결됩니다.

## 2-3. 데이터 전처리 — 방대한 원본에서 신호만 골라내기

원본 국민여행조사 파일은 5만 행, 700개 이상 컬럼의 방대한 규모였습니다. `usecols` 옵션으로 분석에 필요한 열만 선별해 메모리 효율을 확보하고, 헤더 스캔으로 패턴 규칙에 따라 컬럼명을 동적으로 추출해 코드 유지보수성을 높였습니다. 또한 표본가중치(WT_OVER)를 반영한 가중평균으로 집계해 통계적 대표성을 확보했습니다 — 이는 "그럴듯해 보이는 평균"이 아니라 실제 모집단을 대표하는 수치를 쓰겠다는 원칙이었습니다.

# 3. Insight → Decision — "언제"와 "어떻게"는 서로 다른 질문이다

핵심 문제(1절)를 다시 보면 두 개의 서로 다른 질문이 섞여 있습니다: **"언제 환전해야 하는가"**(예측의 문제)와 **"어떻게 환전해야 하는가"**(개인 성향 분류의 문제)입니다. 저희는 이 둘을 하나의 모델로 뭉뚱그리지 않고, 각각 다른 방법론으로 분리해서 풀기로 결정했습니다.

[📷 서비스 파이프라인 다이어그램 — Gemini API(뉴스 요약+영향력점수) + 환율 데이터 → TEMPO → Predict / 사용자 입력 → 국민여행조사 기반 분류 → 환전일·환전 전략 추천]

| 질문 | 담당 모델 | 방법론 |
|---|---|---|
| 언제 환전해야 하는가 | TEMPO 환율 예측 | GPT 기반 시계열 모델 + 뉴스 영향력 지수 |
| 어떻게(일괄/분할) 환전해야 하는가 | 규칙 기반 민감도 분류 | 국민여행조사 기반 점수화 |

# 4. Development — 규칙 기반 비용 민감도 분류 모델

## 4-1. 왜 2개 그룹으로 충분한가 — 임의의 단순화가 아니라 근거 있는 설계

민감도를 고민감/저민감 두 그룹으로만 나누는 것이 성의 없어 보일 수 있습니다. 저희는 이 설계를 정당화하기 위해 명시적으로 검토했습니다: 민감도 연속 점수를 산출한 후 임계값 기반 이진 결정으로 높음/낮음을 분류하는 것은 베이즈 최적 이진 규칙과 DCA(임계확률 기반 의사결정)에 부합하는 통계적으로 타당한 방식이며, 산업·정책 표준도 대부분 2집단으로 시작하고 3집단이 순편익을 유의하게 키우는 증거가 있을 때만 확장하는 방식을 취합니다. **분류 자체를 더 정교하게 만들기보다, 왜 그 정도의 정교함이 충분한지 먼저 검증한 것**입니다.

## 4-2. 점수화 로직 — 국민여행조사 분포를 근거로 가중치를 정하다

```python
# rule_based_classifier/classifier.py
# 점수 계산: 각 요인별로 국민여행조사 데이터 분포를 분석해 부여한 가중치
score_time = 1 if purchase_time in [5, 6] else -1 if purchase_time in [1, 2] else 0
# 여행 임박 시점(1개월/2주 이내)=고민감(+1), 여유 시점(1년/6개월 이내)=저민감(-1)

score_reason = (3 if selection_reasons_input.get(1) == 3 else
                2 if selection_reasons_input.get(2) == 3 else
                1 if selection_reasons_input.get(3) == 3 else 0)
# '저렴한 여행경비'를 선택한 순위에 따라 차등 점수(1순위 +3, 2순위 +2, 3순위 +1)

score_income = 1 if monthly_income in [1, 2, 3] else -1 if monthly_income in [9, 10, 11, 12] else 0
# 월소득 200만원 미만=고민감(+1), 700만원 이상=저민감(-1)

sensitivity_score = score_time + score_reason + score_income
result_group = '고민감 그룹' if sensitivity_score >= 0 else '저민감 그룹'
```

**왜 임의의 숫자가 아니라 분포 기반 점수인가**: 각 항목의 +1/-1/0 임계값(70만원 미만/100만원 이상, 200만원 미만/700만원 이상 등)은 감으로 정한 게 아니라 국민여행조사 데이터의 실제 분포를 분석해 설정했습니다. '저렴한 여행경비'를 선택 이유의 몇 순위에 뒀는지까지 반영해 민감도를 정교하게 측정한 것도, 단일 항목보다 다각도로 검증된 신호가 더 신뢰할 만하다는 판단이었습니다. 최종 민감도 점수가 0점 이상이면 고민감 그룹으로, 여기에 서비스는 "환율이 낮은 날짜에 분할 환전"을, 저민감 그룹에는 "일괄 환전도 무방"이라는 서로 다른 전략을 자동 매칭합니다.

# 5. Development — TEMPO 기반 환율 예측: 뉴스를 어떻게 숫자로 바꿨는가

## 5-1. TEMPO를 선택한 이유

TEMPO(Cao et al., ICLR 2024)는 GPT 기반 시계열 예측 모델로, 시계열을 추세(Trend)·계절성(Seasonality)·잔차(Residual)로 분해해 각 요소를 개별 학습하고, 각 구성요소에 semi-soft prompt를 적용해 learnable vector로 요소별 맥락을 명시적으로 부여하는 두 가지 inductive bias를 가집니다. 환율처럼 추세·계절성이 섞인 데이터에 적합하다고 판단해 채택했습니다.

## 5-2. 텍스트를 시계열에 결합하는 방법 — 논문에 없는 길을 만들다

TEMPO 논문은 TETS(Text for Time Series) 구조로 텍스트 임베딩을 시계열에 결합하는 방법을 제안하지만, **공식 코드에는 멀티모달 입력 기능이 구현돼 있지 않았습니다.** 논문 수준의 멀티모달 입력을 그대로 구현하는 것은 기술적으로 불가능했습니다.

이 지점에서 팀이 내린 결정이 이 프로젝트의 기술적 핵심입니다: 텍스트 임베딩 대신, **뉴스 내용을 수치화한 방향성 지수(impact score)를 입력 변수로 결합하는 방식으로 대체**했습니다.

```python
# src/prepare_input_tensor.py
merged_df["scaled_fx"] = scaler_fx.fit_transform(merged_df[["exchange_rate"]])
merged_df["scaled_impact"] = scaler_impact.fit_transform(merged_df[["impact_score"]])

# 환율 시계열과 뉴스 영향력 지수를 정규화 후 가중합으로 결합
merged_df["combined"] = (
    merged_df["scaled_fx"] * 0.9 +
    merged_df["scaled_impact"] * 0.1
)
```

impact_score는 Google Gemini 2.5 Pro API로 2단계 프롬프팅해서 만들었습니다.

```text
1단계 (뉴스 요약): "Suppose you are living in {year}, can you summarize the key news
events in {year}s {month_name} related to Japan and its impact on the exchange rate
between KRW and JPY? Please directly give me the answer limited to 2 sentences..."

2단계 (영향력 점수화): "Suppose you are a financial analyst. Please rate the impact of
the following news summary on the KRW/JPY exchange rate. Return a single number
between -1.0 and 1.0, where -1.0 means very strong downward pressure..."
```

**왜 2단계로 나눴는가**: 뉴스 원문을 바로 숫자로 평가하게 하면 LLM이 맥락을 압축하지 못한 채 노이즈 섞인 판단을 내릴 위험이 있습니다. 먼저 "2문장 요약"으로 핵심만 추리게 한 뒤, 그 요약문만 가지고 별도 프롬프트로 -1.0~1.0 사이 영향력을 채점하게 함으로써 판단의 근거를 한 단계 명확히 분리했습니다.

## 5-3. alpha 실험 — 감이 아니라 실측으로 가중치를 정하다

뉴스 지수를 얼마나 반영할지(가중치 alpha)를 정하기 위해, 동일한 사전학습 TEMPO 모델로 재학습 없이 alpha만 바꿔가며 2025년 7월 실제 환율과 비교하는 실험을 했습니다.

| Alpha | RMSE | MAE |
|---|---|---|
| 0.00 | 12.9779 | 11.2223 |
| 0.05 | 7.3816 | 5.7494 |
| **0.10** | **5.8541** | **4.7797** |
| 0.15 | 11.1290 | 10.0164 |
| 0.20 | 20.6473 | 20.0815 |
| 0.30 | 42.3564 | 41.8872 |

**왜 이 실험이 중요한가**: alpha=0을 지나 0.10까지는 성능이 개선되다가, 0.15부터는 오히려 급격히 악화됩니다. 뉴스 지수를 "많이 넣을수록 좋다"가 아니라, **적정 수준(10%)에서만 순수 시계열 신호를 보완하고 그 이상은 오히려 노이즈로 작용한다**는 것을 실측으로 확인한 것입니다. 이 실험 없이 임의로 alpha=0.3 같은 값을 썼다면 RMSE가 7배 이상 나빠졌을 것입니다. 최종 서비스에는 alpha=0.1을 적용했습니다.

## 5-4. 예측 → 추천으로 이어지는 마지막 단계

```python
# src/recommend_exchange_dates.py
with torch.no_grad():
    output, _ = model(input_tensor, pred_days, test=True)
    predicted_fx = scaler_fx.inverse_transform(output.reshape(-1, 1)).flatten()

forecast_df = pd.DataFrame({"date": forecast_dates, "predicted_fx": predicted_fx})
# 여행 기간 내 예측 환율이 가장 낮은 top_k일을 추천
recommended = forecast_df.sort_values('predicted_fx').head(top_k).sort_values('date')
```

최종 입력 시계열은 최근 336일 기준으로 구성되며, 부족한 경우 0-padding을 적용해 (1, 336, 1) 형식의 PyTorch 텐서로 변환한 뒤 TEMPO 모델에 입력했습니다. 사용자가 입력한 출국일·입국일 범위 내에서 예측 환율이 가장 낮은 날짜를 정렬해 환전 추천일로 제시합니다.

# 6. Service Design — Streamlit으로 완성한 사용자 여정

[📷 ¥ETI 주요 서비스 화면 — 환율 현황 카드, 환율 추이 및 예측 그래프, 환전 시점 추천]

Streamlit 기반 웹 서비스는 4가지 기능으로 구성됩니다.

1. **환율 변동 정보 요약**: 당일 환율 현황, 내일 환율 예측, 여행 기간 전 최저 예상 환율을 카드 형태로 제공. 과거 데이터와 AI 예측 데이터를 한눈에 비교할 수 있는 그래프 포함
2. **예측 환율 기반 환전 시점 추천**: 여행 기간과 금액을 입력하면 환율이 가장 낮을 것으로 예상되는 일자와 예상 환전액을 제안
3. **사용자 맞춤형 스마트 환전 플래너**: 4문항 설문 결과를 바탕으로 분할/일괄 환전 전략을 추천
4. **월별 리포트**: 최근 3개월의 AI 뉴스 요약을 제공해 엔화 시장 흐름을 사용자가 직접 확인 가능

> "AI가 추천하는 환전 시점 알아보기"라는 홈 화면 문구가 상징하듯, 이 서비스는 "환율이 얼마다"라는 정보 제공에 그치지 않고 "당신은 이런 사람이니 이렇게 하라"는 결론까지 내려주는 것을 목표로 설계했습니다.

# 8. Reflection — 발전 가능성과 한계점

- **자동화 수준의 한계**: 현재 제출된 소스코드는 환율 데이터를 사용자가 원할 때 수동으로 업데이트해야 하는 구조입니다. 한국은행 ECOS API를 연동해 실시간 환율 데이터를 자동으로 수집·반영하도록 개선하면 파이프라인의 시의성이 크게 향상될 것입니다.
- **무료 API의 한계**: 뉴스 요약 및 영향력 점수 생성에 현재 무료 Google Gemini API를 사용하고 있어 요청 제한 및 응답 지연이 발생할 수 있습니다. 유료 API 전환 시 처리 속도와 신뢰도를 높일 수 있습니다.
- **통화 확장성**: 현재는 원/엔화 중심으로 구현했으나, 현금 사용률이 높은 국가(태국, 베트남, 대만 등)로 확장하면 다양한 해외여행자에게 적용할 수 있습니다.
- **데이터 정밀도**: 카드사·여행사와 연계해 실제 결제 이력 기반 고객 세분화를 수행하면, 4문항 규칙 기반 분류보다 더 정밀한 맞춤형 환전 전략이 가능해지고 금융기관과의 B2B 제휴로도 확장할 수 있습니다.

> TEMPO 공식 코드에 없는 멀티모달 입력을 그대로 포기하지 않고, "텍스트 임베딩 대신 방향성 지수로 대체한다"는 실용적 우회로를 찾은 것과, 그 대체 방식의 가중치(alpha)를 감이 아니라 6개 지점 실측으로 검증한 것 — 이 두 가지가 짧은 공모전 기간 안에서 "논문을 그대로 베끼지 않고 우리 문제에 맞게 재구성했다"고 자신 있게 말할 수 있는 지점입니다.
